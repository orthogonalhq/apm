## Overview

APM uses two project-root files to bridge installed skills with AI agents:

| File | Owner | Purpose |
|------|-------|---------|
| `SKILLS.md` | APM (auto-generated) | Index of all installed skills with metadata |
| `AGENTS.md` | User (created once by APM) | Universal agent entry point that references skills |

Together, these files provide **discovery** (agents know what skills are available) and **disclosure** (users can see exactly what instructions their agent is operating under).

## SKILLS.md

`SKILLS.md` is auto-generated at the project root by `apm install`, `apm uninstall`, and `apm update`. It is a human-readable, agent-parseable index of every installed skill.

**You should not edit this file manually.** APM regenerates it from the lockfile and installed SKILL.md frontmatter on every operation.

### Format

```markdown
# Installed Skills

<!-- This file is auto-generated by APM. Do not edit manually. -->

## @anthropics/code-review

- **Kind:** skill
- **Description:** Review code changes for security, performance, and correctness.
- **Tags:** code-review, security, performance
- **Path:** .skills/anthropics/code-review/SKILL.md

## @cloudflare/code-review

- **Kind:** skill
- **Description:** Reviews Workers and Cloudflare Developer Platform code.
- **Tags:** cloudflare, workers, typescript
- **Path:** .skills/cloudflare/code-review/SKILL.md
```

### Fields

Each skill entry includes:

| Field | Source | Description |
|-------|--------|-------------|
| **Kind** | `kind` frontmatter field | One of: `skill`, `composite-skill`, `workflow`, `app` |
| **Description** | `description` frontmatter field | One-line description from the SKILL.md |
| **Tags** | `tags` frontmatter field | Comma-separated list of tags |
| **Path** | Derived from scope/name | Relative path to the installed SKILL.md |

### When it updates

- `apm install @scope/name` — adds the new entry
- `apm install` (no args) — regenerates from lockfile
- `apm uninstall @scope/name` — removes the entry
- `apm update` — refreshes all entries with latest metadata

When all skills are uninstalled, `SKILLS.md` is removed.

## AGENTS.md

`AGENTS.md` is the universal entry point for AI agents. It lives at the project root and tells agents where to find skill definitions.

APM creates `AGENTS.md` on the first `apm install` if it doesn't exist. If it already exists, APM injects a reference block without modifying existing content.

### Generated AGENTS.md

When no `AGENTS.md` exists, APM creates:

```markdown
# Agent Configuration

<!-- apm:skills -->
This project uses [APM](https://apm.orthg.nl) to manage agent skills.
See [SKILLS.md](./SKILLS.md) for all installed skills and their descriptions.
<!-- /apm:skills -->
```

### Injected block

When `AGENTS.md` already exists (e.g. with your own project instructions), APM appends:

```markdown
<!-- apm:skills -->
This project uses [APM](https://apm.orthg.nl) to manage agent skills.
See [SKILLS.md](./SKILLS.md) for all installed skills and their descriptions.
<!-- /apm:skills -->
```

The `<!-- apm:skills -->` markers let APM identify its own section. APM will never modify content outside these markers.

### User content

You are encouraged to add your own project-specific agent instructions to `AGENTS.md`:

```markdown
# Agent Configuration

## Project context

This is a Next.js e-commerce application. The API is in /app/api.
Always use server components unless client interactivity is needed.

## Coding standards

- Use TypeScript strict mode
- Prefer named exports
- Keep components under 200 lines

<!-- apm:skills -->
This project uses [APM](https://apm.orthg.nl) to manage agent skills.
See [SKILLS.md](./SKILLS.md) for all installed skills and their descriptions.
<!-- /apm:skills -->
```

## How agents use these files

### Agent discovery flow

1. Agent starts in a project directory
2. Agent reads `AGENTS.md` at the project root
3. `AGENTS.md` references `SKILLS.md`
4. Agent reads `SKILLS.md` to discover available skills
5. Agent reads individual `SKILL.md` files as needed (paths listed in `SKILLS.md`)

### Compatibility

Since `AGENTS.md` and `SKILLS.md` are plain markdown, they work with any agent that reads project files:

| Agent | How to point it at AGENTS.md |
|-------|------------------------------|
| Claude Code | Add `See AGENTS.md` to your `CLAUDE.md` |
| Cursor | Reference in `.cursorrules` |
| GitHub Copilot | Reference in `.github/copilot-instructions.md` |
| Gemini CLI | Reference in `GEMINI.md` |
| Any agent | The agent reads `AGENTS.md` by convention |

As more agents adopt `AGENTS.md` as a standard, explicit references will become unnecessary.

## Version control

| File | Commit? | Reason |
|------|---------|--------|
| `AGENTS.md` | Yes | User-owned configuration |
| `SKILLS.md` | Yes | Reproducible skill discovery for the team |
| `apm-lock.json` | Yes | Reproducible installs |
| `.skills/` | Optional | Some teams commit skills, others regenerate from lockfile |

## Project structure

After installing skills, your project root looks like:

```
your-project/
├── AGENTS.md              # Agent entry point (user-owned)
├── SKILLS.md              # Skill index (auto-generated)
├── apm-lock.json          # Lockfile
├── .skills/
│   ├── anthropics/
│   │   └── code-review/
│   │       └── SKILL.md
│   └── cloudflare/
│       └── code-review/
│           └── SKILL.md
└── ... your project files
```
