## Package names

All packages use scoped names in the format `@scope/name`. The scope is typically the GitHub owner or organization:

```
@anthropics/code-review
@user/my-skill
@myorg/internal-guidelines
```

## `apm init`

Set up APM for your project. Interactive, guided setup that creates agent configuration files and wires up skill references.

```bash
apm init
```

**What it does:**

1. Creates `AGENTS.md` at project root if it doesn't exist
2. Lists your installed skills with descriptions
3. Adds `@skill:scope/name` references to `AGENTS.md`
4. Detects agent config files (`CLAUDE.md`, `.cursorrules`, `.github/copilot-instructions.md`, `GEMINI.md`, `.windsurfrules`) and offers to link them to `AGENTS.md`

Every step prompts for confirmation and explains what it's doing and why. Safe to run multiple times вҖ” it detects existing state and skips.

**Example:**

```bash
$ apm init

apm Setting up APM for this project.

  ? Create AGENTS.md? (Y/n) y
    AGENTS.md is the entry point for AI agents working in your project.
    It contains project-specific instructions and references to skills.
  вң“ Created AGENTS.md

  You have 2 installed skill(s):
    @anthropics/code-review вҖ” Review code changes for security...
    @cloudflare/code-review вҖ” Reviews Workers and Cloudflare...

  ? Add skill references to AGENTS.md? (Y/n) y
    @skill:scope/name tells agents which installed skills to load.
    Agents read these references and load the corresponding SKILL.md files.
  вң“ Added @skill references to AGENTS.md

  Detected CLAUDE.md
  ? Add a reference to AGENTS.md in CLAUDE.md? (Y/n) y
    This tells Claude Code to read AGENTS.md, which links to your skills.
  вң“ Updated CLAUDE.md

apm Done! Your agents can now discover installed skills.
  Learn more: https://apm.orthg.nl/docs/skill-references
```

See [Skill References](/docs/skill-references) for the full spec on the `@skill:` convention.

## `apm install` (alias: `apm i`)

Install a skill by scoped name, or restore all from lockfile.

```bash
# Install a single skill
apm install @scope/name

# Restore all skills from lockfile
apm install
```

**What it does:**

1. Fetches package metadata from the registry
2. Creates `.skills/scope/name/` in your project root
3. Writes the `SKILL.md` file
4. Updates `apm-lock.json` with source info, description, kind, and tags

When run without arguments, re-fetches and installs all packages listed in `apm-lock.json`. Useful for cloning a project and restoring its skills.

**Project root detection:** APM walks up from the current directory looking for `apm-lock.json`, `package.json`, `Cargo.toml`, or `.git`. Falls back to the current directory.

**Example:**

```bash
$ apm install @anthropics/code-review
apm Fetching @anthropics/code-review...
apm Installed @anthropics/code-review to .skills/anthropics/code-review
```

**Result:**

```
your-project/
в”ңв”Җв”Җ apm-lock.json
в””в”Җв”Җ .skills/
    в””в”Җв”Җ anthropics/
        в””в”Җв”Җ code-review/
            в””в”Җв”Җ SKILL.md
```

After installing, run `apm init` to wire skills into your agent configs.

## `apm uninstall` (alias: `apm u`)

Remove an installed skill.

```bash
apm uninstall @scope/name
```

**What it does:**

1. Removes `.skills/scope/name/` directory
2. Cleans up empty scope directories
3. Removes the entry from `apm-lock.json`

**Example:**

```bash
$ apm uninstall @anthropics/code-review
apm Uninstalled @anthropics/code-review
```

## `apm update` (alias: `apm up`)

Re-fetch latest versions for all installed skills.

```bash
apm update
```

**What it does:**

1. Reads all packages from `apm-lock.json`
2. Fetches the latest version of each from the registry
3. Overwrites the local `SKILL.md` files
4. Updates the lockfile with latest metadata

**Example:**

```bash
$ apm update
apm Updating 3 package(s)...
  вҶ‘ @anthropics/code-review... вң“
  вҶ‘ @cloudflare/code-review... вң“
  вҶ‘ @microsoft/code-review... вң“
apm All packages updated
```

## `apm search`

Search for skills in the registry.

```bash
apm search <query>
```

Searches the registry using full-text search over scopes, names, and descriptions. Results show the scoped name, metadata (stars, license), description, and source repository.

**Example:**

```bash
$ apm search "error handling"
Found 3 skill(s) matching 'error handling':

  @user/error-handling (вҳ… 42 В· MIT)

  user/error-handling-skill

  @org/resilient-apis (вҳ… 18 В· Apache-2.0)
```

## `apm validate`

Validate a local SKILL.md file against the specification.

```bash
apm validate <path>
```

Accepts either a path to a `SKILL.md` file or a directory containing one. Checks:

- YAML frontmatter is present and parseable
- `name` field exists, is valid (lowercase alphanumeric + hyphens, max 64 chars, no consecutive hyphens)
- `name` matches the parent directory name
- `description` field exists and is within 1024 characters

**Exit codes:**

| Code | Meaning |
|------|---------|
| `0` | Valid |
| `1` | Validation errors found |

**Example (valid):**

```bash
$ apm validate ./my-skill
вң“ SKILL.md is valid
  name: my-skill
  description: A description of what this skill does
```

**Example (invalid):**

```bash
$ apm validate ./bad-skill
вң— SKILL.md has 2 error(s):
  В· name must be lowercase alphanumeric with hyphens
  В· description must not be empty
```

## Lockfile

APM maintains an `apm-lock.json` file at your project root. It serves two purposes:

1. **Reproducible installs** вҖ” run `apm install` on a fresh clone to restore all skills
2. **Skill index** вҖ” metadata for tooling, editors, and agents to understand what's installed

Since lockfile v2, each entry includes metadata from the SKILL.md frontmatter:

```json
{
  "lockfileVersion": 2,
  "packages": {
    "@anthropics/code-review": {
      "sourceRepo": "anthropics/knowledge-work-plugins",
      "sourcePath": "engineering/skills/code-review/SKILL.md",
      "sourceRef": "main",
      "commitSha": null,
      "description": "Review code changes for security, performance, and correctness.",
      "kind": "skill",
      "tags": ["code-review", "security"]
    }
  }
}
```

| Field | Description |
|-------|-------------|
| `sourceRepo` | GitHub repository |
| `sourcePath` | Path to SKILL.md within the repo |
| `sourceRef` | Git branch or tag |
| `commitSha` | Pinned commit (when available) |
| `description` | One-line description from frontmatter |
| `kind` | `skill`, `composite-skill`, `workflow`, or `app` |
| `tags` | Tags from frontmatter |

Commit `apm-lock.json` to version control.
