## What is APM?

APM (Agent Package Manager) is the package manager and public registry for agent skills. It connects skill authors to the 34+ AI agent products that support the [agentskills.io](https://agentskills.io) open standard — including Claude Code, Cursor, GitHub Copilot, Gemini CLI, Windsurf, and more.

All packages use scoped names (`@scope/name`) to avoid collisions and provide clear provenance. The scope is derived from the GitHub repo owner for indexed packages.

Discover skills on the [registry](https://apm.orthg.nl), install them with one command, and tell your agent which skills to use.

## Install the CLI

### npm

```bash
npm install -g @apm-cli/apm
```

### Homebrew (macOS / Linux)

```bash
brew install orthogonalhq/apm/apm
```

### Shell (macOS / Linux)

```bash
curl -fsSL https://apm.orthg.nl/install.sh | sh
```

### Windows

```bash
winget install Orthogonal.APM
```

### From source

```bash
git clone https://github.com/orthogonalhq/apm.git
cd apm/cli
cargo build --release
```

## Quick start

### 1. Search for skills

```bash
apm search "code review"
```

Results show the scoped name, stars, license, and description for each match:

```
Found 10 skill(s) matching 'code review':

  @anthropics/code-review (★ 9330 · Apache-2.0)
  Review code changes for security, performance, and correctness.
  anthropics/knowledge-work-plugins

  @cloudflare/code-review (★ 4480 · CC-BY-4.0)
  Reviews Workers and Cloudflare Developer Platform code.
  cloudflare/cloudflare-docs
  ...
```

<Callout>
  Use the [APM Registry](https://apm.orthg.nl/packages) for browsing skills with filters, sorting, and full-text search across 10,000+ indexed packages.
</Callout>

### 2. Install a skill

```bash
apm install @anthropics/code-review
```

APM fetches the skill from the registry and writes it to your project:

```
your-project/
├── apm-lock.json
└── .skills/
    └── anthropics/
        └── code-review/
            └── SKILL.md
```

### 3. Set up agent discovery

```bash
apm init
```

This walks you through creating an `AGENTS.md` file at your project root and adding `@skill:` references so your agent knows which skills to load. Every step explains what it's doing and asks for confirmation.

After setup, your `AGENTS.md` looks like:

```markdown
# Agent Configuration

<!-- project-specific instructions here -->

<!-- apm:skills -->
## Installed Skills

@skill:anthropics/code-review

<!-- /apm:skills -->
```

### 4. Reference from your agent config

Point your agent at `AGENTS.md`:

| Agent | Config file | What to add |
|-------|-------------|-------------|
| Claude Code | `CLAUDE.md` | `See AGENTS.md for installed agent skills.` |
| Cursor | `.cursorrules` | `See AGENTS.md for installed agent skills.` |
| GitHub Copilot | `.github/copilot-instructions.md` | `See AGENTS.md for installed agent skills.` |
| Gemini CLI | `GEMINI.md` | `See AGENTS.md for installed agent skills.` |

`apm init` can do this for you — it detects your agent config files and offers to add the reference.

### 5. Manage your skills

```bash
# Update all skills to latest
apm update

# Remove a skill
apm uninstall @anthropics/code-review

# Restore skills on a fresh clone
apm install
```

## How it works

1. **Discover** — browse [apm.orthg.nl](https://apm.orthg.nl) or use `apm search`
2. **Install** — `apm install @scope/name` fetches the skill and writes it to `.skills/`
3. **Wire up** — `apm init` creates `AGENTS.md` and links your agent configs to installed skills
4. **Use** — your agent reads `AGENTS.md`, sees `@skill:` references, and loads the corresponding SKILL.md files

APM tracks installed packages in `apm-lock.json` — a lockfile that includes source info, descriptions, tags, and kind for each skill. Commit it to version control for reproducible installs.
